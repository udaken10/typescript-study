# 01. TypeScriptのキホン（変数と型）

こんにちは！TypeScriptの世界へようこそ。この章では、プログラミングの基本である「変数」と、TypeScriptの最大の特徴である「型（かた）」について学びます。

## 1. 変数は「箱」、型は「ラベル」
プログラミングでは、データ（名前や数字など）を入れておくための「箱」を作ります。これを**変数**と呼びます。

JavaScriptではどんな箱にでも好きなものを入れられましたが、TypeScriptでは箱に**「これは数字専用！」「これは文字専用！」というラベル**を貼ります。これが**型**です。

### なぜラベル（型）が必要なの？
例えば、「料理の材料」を入れる箱に間違えて「洗剤」を入れてしまったら大変ですよね？
型を決めておくと、プログラムを実行する前に「あ！その箱にそれは入れちゃダメだよ！」とエディタが教えてくれるので、ミスを未然に防げます。

## 2. よく使う型（プリミティブ型）
高校の授業や日常でよく使うデータをイメージしてみましょう。

- `string`（ストリング）: **文字列**。名前やメッセージなど。`"`（ダブルクォート）で囲みます。
- `number`（ナンバー）: **数値**。テストの点数や身長など。
- `boolean`（ブーリアン）: **真偽値**。`true`（はい）か `false`（いいえ）の2つだけ。

```typescript
let studentName: string = "たろう";
let mathScore: number = 85;
let isPassed: boolean = true;
```

## 3. ちょっと特殊な型
- `any`（エニー）: **なんでも入る魔法の箱**。便利そうですが、こればかり使うとTypeScriptを使う意味がなくなってしまうので、なるべく避けましょう。
- `void`（ヴォイド）: **「空っぽ」という意味**。主に「値を返さない関数（命令）」で使います。

## 4. 型推論（かたすいろん）
TypeScriptは賢いので、最初に値を入れると「あ、この箱は文字列用だね！」と自動で判断してくれます。これを**型推論**と呼びます。

```typescript
let message = "こんにちは"; // 最初が文字なので、自動的に string 型になる
// message = 100; // これはエラーになる！文字の箱に数字は入れられないから。
```
